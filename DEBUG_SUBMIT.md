# 🔍 訊號提交問題排查

## 請立即執行以下檢查：

### 步驟 1：查看瀏覽器 Console
請告訴我 Console 中顯示了什麼訊息？

特別注意：
- 是否有紅色錯誤訊息？
- 是否顯示「🚀 開始發布訊號」？
- 是否顯示「✅ 發布成功」或「❌ 發布失敗」？

### 步驟 2：檢查 Network 請求
1. 打開 DevTools > Network 面板
2. 發布一個訊號
3. 查看是否有 `/api/signals` 的 POST 請求
4. 點擊該請求，查看：
   - Status Code 是多少？
   - Response 內容是什麼？

### 步驟 3：檢查訊號列表
發布後：
- 頁面是否自動刷新？
- 列表數量是否有變化？
- Console 是否顯示成功訊息但列表沒更新？

---

## 可能的原因

### 原因 1：驗證失敗
如果 Console 顯示驗證錯誤，可能是：
- 電影沒有選擇
- 標籤沒有選擇

### 原因 2：API 請求失敗
如果 Network 顯示錯誤狀態碼：
- 401: 認證問題
- 500: 伺服器錯誤

### 原因 3：列表未刷新
如果 Console 顯示成功但列表沒更新：
- 可能是 useSignals 的 fetchSignals 沒有被調用
- 可能是篩選條件導致訊號被過濾掉
