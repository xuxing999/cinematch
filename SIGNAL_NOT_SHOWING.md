# 🔍 訊號發布成功但不顯示 - 排查指南

## 問題症狀

✅ Console 顯示發布成功
✅ 訊號數據完整
❌ 頁面上訊號數量顯示 0

---

## 最可能的原因：篩選條件

### 請立即檢查：

**查看頁面上方的篩選按鈕**

```
[全部] [🎟️我有票] [👻求壯膽] [🎭純看片] [🧠想討論]
```

**請告訴我：**
- 哪個按鈕是紅色高亮的？
- 您發布的訊號標籤是：`has_ticket`（我有票）

**如果選擇的不是「全部」或「我有票」，訊號就不會顯示！**

---

## 解決方案

### 方案 1：點擊「全部」按鈕
點擊頁面上方的 **[全部]** 按鈕，應該就能看到訊號了。

### 方案 2：點擊「我有票」按鈕
點擊 **[🎟️ 我有票]** 按鈕，也應該能看到訊號。

---

## 驗證訊號是否真的存在

請在 Console 中執行：

```javascript
fetch('/api/signals').then(r => r.json()).then(data => {
  console.log('所有訊號:', data)
  console.log('訊號數量:', data.length)
})
```

應該會顯示至少 1 個訊號。

---

## 如果切換篩選後仍然看不到

請告訴我：
1. 點擊「全部」後，訊號數量是多少？
2. Console 執行上面的代碼後，返回多少個訊號？
3. 訊號的 `created_at` 時間是什麼？（可能是 24 小時過濾的問題）
